# Introduction-to-Artificial-Intelligence-hw
同济大学软件工程专业人工智能导论课程第三次作业——大语言模型部署体验

## 一、项目简介

本项目基于魔搭平台，通过Jupyter Notebook完成了两个开源大语言模型 **Qwen-7B-Chat** 和 **ChatGLM3-6B** 的本地部署，并基于 Python 编写了一个终端交互程序，实现了本地模型对话、上下文问答以及不同模型之间的动态切换功能。

项目主要围绕以下内容展开：

1. 完成 Qwen-7B-Chat 和 ChatGLM3-6B 的本地部署；
2. 编写终端对话程序，实现模型切换和多轮问答；
3. 使用相同问题对两个模型进行问答测试；
4. 从部署难度、运行性能、中文表达能力、逻辑理解能力和资源占用等方面对两个模型进行横向对比分析。

---

## 二、项目环境

本项目最初使用云端 CPU 环境进行测试：

```text
CPU 环境：8 核 32GB
```

在 CPU 环境下，ChatGLM3-6B 可以完成基本对话任务，但 Qwen-7B-Chat 生成速度极慢，并且容易出现内存压力。因此后续切换到 GPU 环境进行测试：

```text
GPU 环境：8 核 32GB，显存 24GB
```

在 GPU 环境下，两个模型均能成功加载并完成正常问答，生成速度明显提升。

## 三、模型部署

### 3.1 Qwen-7B-Chat

Qwen-7B-Chat 是阿里云通义千问团队开源的对话模型，具有较强的中文问答、结构化表达和复杂指令理解能力。

本项目将其部署在本地路径下：

```text
/mnt/workspace/Qwen-7B-Chat
```

部署完成后，可通过程序中的模型切换命令加载该模型进行问答。

### 3.2 ChatGLM3-6B

ChatGLM3-6B 是智谱 AI 和清华 KEG 开源的中英双语对话模型，部署门槛相对较低，对中文对话任务支持较好。

本项目将其部署在本地路径下：

```text
/mnt/workspace/chatglm3-6b
```

部署完成后，可通过程序中的模型切换命令加载该模型进行问答。

## 四、项目功能

本项目编写了一个终端对话程序，用于实现本地模型的调用和切换。程序支持以下功能：

```text
/models              查看当前支持的所有模型
/switch GLM          切换到 ChatGLM3-6B
/switch Qwen         切换到 Qwen-7B-Chat
/clear               清空当前模型的对话历史
/clear_all           清空所有模型的对话历史
/history             查看当前模型的历史对话
/help                查看帮助信息
/exit                退出程序
```

程序运行后，可以在终端中输入问题，并根据当前加载的模型返回回答。

## 五、运行方式

```bash
conda create -n qwen_env python=3.10 -y
conda activate qwen_env
```

### 5.2 安装依赖

```bash

```

如果在 GPU 环境下运行，需要确保 PyTorch 可以正常识别 GPU：

```bash
python -c "import torch; print(torch.cuda.is_available())"
```

如果输出结果为：

```text
True
```

说明 GPU 可用。

```bash
python local_chat.py
```

程序启动后，可以通过以下命令切换模型：

```text
/switch GLM
```

或：

```text
/switch Qwen
```

随后即可在终端中输入问题，进行本地对话测试。

## 七、模型横向对比分析

### 7.1 基本信息对比

| 对比维度 | ChatGLM3-6B | Qwen-7B-Chat |
|---|---|---|
| 模型规模 | 约 6B 参数级别 | 约 7B 参数级别 |
| 模型定位 | 面向中英双语对话与本地部署 | 面向通用对话、知识问答、代码与推理任务 |
| 开源机构 | 智谱 AI / 清华 KEG | 阿里云通义千问团队 |
| 上下文长度 | 支持较长上下文，ChatGLM3-6B 支持 8K | 支持较长上下文，Qwen-7B 更新后可支持 8192 上下文长度 |
| 调用方式 | AutoModel + model.chat() | AutoModelForCausalLM + model.chat() |
| 部署特点 | 部署门槛相对较低，对中文对话支持较好 | 综合能力较强，但资源占用更高 |
| 本地 CPU 表现 | 相对更容易跑通 | 运行较慢，容易出现长时间等待或内存压力 |
| 本地 GPU 表现 | 24GB 显存下运行较稳定 | 24GB 显存下可以运行，但加载和生成资源占用更高 |

从基本信息来看，ChatGLM3-6B 的模型规模略小，部署门槛相对较低，更适合资源有限环境下的本地测试。Qwen-7B-Chat 的模型规模更大，综合能力更强，但对硬件资源要求也更高。

### 7.2 对话效果对比

经过多轮对话测试后可以发现，两个模型均具备较好的中文对话能力，但在回答风格和复杂问题处理能力方面存在差异。

ChatGLM3-6B 的回答较为流畅，能够较好完成日常问答、简单解释和基础对话任务，整体风格偏简洁直接。但对于一些逻辑性较强、语言歧义较多的问题，ChatGLM3-6B 有时表现不够稳定，容易出现理解偏差或回答与问题不完全对应的情况。

Qwen-7B-Chat 的回答通常更加完整，尤其在需要分点说明、概念解释和文本整理的任务中，内容组织性更强，回答信息量也更大。在复杂逻辑问题和语言理解问题上，Qwen-7B-Chat 的表现相对更好。不过，由于其输出内容通常更长，在本地部署时也会带来更高的推理开销。

| 对比维度 | ChatGLM3-6B | Qwen-7B-Chat |
|---|---|---|
| 中文表达 | 表达自然，回答比较稳 | 表达更完整，内容组织性较强 |
| 指令理解 | 对普通问答和简单任务理解较好 | 对复杂指令、分点回答、解释类问题更有优势 |
| 回答风格 | 简洁直接，适合快速问答 | 内容更丰富，适合生成解释性文本 |
| 多轮对话 | 多轮对话较稳定 | 多轮对话能力较好，但上下文越长资源消耗越明显 |
| 幻觉风险 | 仍可能出现不准确回答 | 也可能出现编造内容，需要人工校验 |
| 逻辑能力 | 逻辑理解能力一般，复杂问题处理不够稳定 | 逻辑理解能力较强，能较好处理复杂逻辑问题 |

### 7.3 性能及资源占用对比

从运行性能来看，ChatGLM3-6B 在本地环境中的部署适应性更好。在 CPU 环境下，ChatGLM3-6B 虽然响应速度有限，但基本能够完成对话；而 Qwen-7B-Chat 对计算资源更加敏感，在 CPU 环境下生成速度很慢，并且容易因为模型权重和推理缓存占用较大而导致内存不足。

切换到 GPU 环境后，两个模型均可获得明显加速，能够完成正常问答。但 Qwen-7B-Chat 对显存和 CUDA 环境的要求更高，部署时需要注意 PyTorch、CUDA 与显卡驱动版本是否匹配。

| 对比维度 | ChatGLM3-6B | Qwen-7B-Chat |
|---|---|---|
| CPU 运行 | 可以运行，但速度有限 | 非常慢，容易卡顿或内存不足 |
| GPU 运行 | 运行较稳定 | 运行明显改善，但显存占用更高 |
| 加载速度 | 相对较快 | 相对较慢 |
| 内存占用 | 较低 | 较高 |
| 适合场景 | 轻量部署、中文对话、教学演示 | 更复杂问答、代码、推理和文本生成 |

## 八、实验总结

本项目对 ChatGLM3-6B 与 Qwen-7B-Chat 两个开源大语言模型进行了云端部署和对话测试。实验结果表明，两个模型均支持中文对话任务，但在部署成本、运行效率和回答质量方面存在一定差异。

ChatGLM3-6B 的主要优势在于部署门槛较低、中文对话流畅、资源占用相对较小，在 CPU 环境下也能完成基本对话任务，因此更适合本地轻量级实验、入门学习测试和教学演示场景。

Qwen-7B-Chat 的模型规模更大，综合能力更强，在复杂指令理解、结构化回答和逻辑分析等方面具有更高潜力。但与此同时，它对硬件资源要求更高，在 CPU 环境下响应速度较慢，容易出现长时间等待和内存压力；切换到 GPU 环境后，其运行效率明显提升，更适合在 GPU 资源充足的环境中承担复杂问答和高质量文本生成任务。

综合来看，如果项目目标是实现稳定、易部署的本地对话系统，可以优先选择 ChatGLM3-6B；如果项目更加关注回答质量、复杂任务处理能力和模型能力上限，并且具备较好的 GPU 资源，则 Qwen-7B-Chat 更具优势。
